This index tracks module hardening status and links to the per-module records.

## Policy (Applies to All New Features)

- Every new feature or module must follow `security/HARDENING_CHECKLIST.md`.
- Add or update the corresponding `security/hardening/*.md` entry.
- Record verification evidence (tests or artifacts) in the module doc.
