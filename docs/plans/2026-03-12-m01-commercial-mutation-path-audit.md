# M01 Commercial Mutation Path Audit

## Scope

Audit the six commercial mutation surfaces named by `M01` and classify each one by canonical entrypoint, auth or tenant boundary, idempotency posture, audit surface, and supersession target. This audit does not change routing, auth, tenancy, or `apps/web/src/proxy.ts`.

## Key Findings

- Only two of the six named `M01` surfaces are already canonical typed server mutations: agreement acceptance and cancellation.
- `Escalation request` is not a durable commercial mutation today. It is a telemetry side effect attached to claim submission.
- `Free Start submission` is also not a durable mutation today. The public shell completes locally and emits analytics only.
- `Matter consumption` is still a gap. The repo enforces agreement prerequisites before staff-led recovery starts, but it does not yet decrement an allowance ledger or record matter consumption as its own mutation.
- `Roster import` is also still a gap. The current agent import UI is a client-only placeholder and does not satisfy the group-access roster import called for by the blueprint.
