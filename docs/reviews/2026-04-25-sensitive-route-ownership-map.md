# 2026-04-25 Sensitive Route Ownership Map

## Purpose

This map closes `P17-PR04` by naming the owner, boundary contract, and current proof
surface for sensitive app routes. It is an execution aid for future bounded pilot work, not
an architecture redesign.

The repo-canonical constraints still apply:

- `apps/web/src/proxy.ts` remains the only routing and access-control authority.
- Canonical routes remain `/member`, `/agent`, `/staff`, and `/admin`.
- Supabase Auth remains the identity system of record, with better-auth orchestrating the
  active session boundary through `@interdomestik/shared-auth`.
- Stripe is not part of V3 pilot flows.
- Any uncovered item below is a future candidate unless a later slice explicitly promotes
  it.

## Current Gap Notes

These notes are intentionally not fixed in this slice.

1. The route-level verification API currently has core tests for role and not-found behavior, but no focused route test that asserts missing session, malformed tenant identity, or branch scoping behavior at the API boundary.
2. Share-pack POST is protected by session and service-level tenant filtering, but the route does not yet have a focused tenant-missing negative test comparable to the AI/settings routes hardened in `P17-PR02`.
3. The tracker proof ledger remains a large Markdown table. It should be split only after a separate generated-ledger or append-only evidence design is approved.

## Expected Use

Future sensitive-route slices should update this map when they materially change an owner,
entrypoint, boundary helper, failure contract, or proof file. Small implementation-only
bug fixes do not need to change this file unless they alter one of those named contracts.
